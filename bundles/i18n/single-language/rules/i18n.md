# i18n: Single Language

## What pattern is used

This project has no translation/internationalization tooling. All UI text is written
directly in templates and component code, in the one language this app is built for.

## What the AI agent may do

- Write display text directly in templates as normal.

## What the AI agent must NOT do

- Do not add a translation library (Transloco, ngx-translate, `@angular/localize`, etc.)
  "just in case." If multi-language support becomes a real requirement, that's a
  bundle-selection change for a developer to make (re-running the generator's `i18n`
  axis as `multi-language`), not something to bolt on mid-task.
- Do not create a translation-keys JSON structure speculatively — there's nothing to
  load it with in this project.

## Where the code lives

N/A — no i18n code in this project by design.
