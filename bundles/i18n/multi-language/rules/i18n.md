# i18n: Multi-Language (Transloco)

## What pattern is used

This project supports runtime language switching via Transloco — added through its own
CLI schematic (`ng add @jsverse/transloco`), not hand-written, so the setup always
matches whatever version is actually installed. Translation strings live in JSON files
under `public/i18n/<lang>.json`. `transloco.config.ts` (project root) lists the
supported languages. The loader (`src/app/transloco-loader.ts`) fetches the right JSON
file at runtime — no rebuild needed to add or switch a language.

## What the AI agent may do

- **Add new translation keys to `public/i18n/<lang>.json` for every supported
  language** whenever adding new UI text — these files are expected to grow constantly
  and are not protected.
- Use Transloco's `t` pipe (`{{ 'key.path' | transloco }}`) or directive in templates,
  or the `TranslocoService`/`translate()` function in component code, for any
  user-facing text — never hardcode display strings when this bundle is selected.
- Add a new supported language by adding it to `transloco.config.ts`'s `langs` array
  (this file is protected from *automatic* edits — see below — but the *language list*
  itself is exactly the kind of change a developer should review, since it also means
  someone needs to actually provide translations for every existing key).

## What the AI agent must NOT do

- **Do not edit** `transloco.config.ts` or `src/app/transloco-loader.ts` — protected by
  `.claude/settings.json`. If a new language needs to be added, flag it — someone still
  needs to supply the actual translations for every key, which isn't something to
  auto-generate.
- Do not hardcode any user-facing string directly in a template or component once this
  bundle is selected — that string won't be translatable, defeating the point.
- Do not add a second translation library (ngx-translate, `@angular/localize`, etc.)
  alongside Transloco.
- Do not guess a translation for a language you don't have real content for — leave a
  clear placeholder or flag the missing key rather than inventing text in a language the
  agent doesn't actually know is correct.

## Where the code lives

- `transloco.config.ts` (project root) — supported languages list (protected).
- `src/app/transloco-loader.ts` — fetches translation JSON at runtime (protected).
- `public/i18n/<lang>.json` — the actual translation strings (editable, expected to grow).
- `src/app/app.config.ts` — already wired with `provideTransloco(...)` by the schematic;
  this file remains generally editable for unrelated app wiring, but don't remove or
  duplicate the Transloco provider entry.
